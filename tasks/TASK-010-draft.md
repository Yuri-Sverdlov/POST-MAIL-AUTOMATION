## BUG: Мусор после strip_html()

**Что убрать:**
- `&zwnj;`, `&zwsp;`, `&#8202;` и любые zero-width символы
- Артефакт `96` в начале тела писем
- `\r\n` → `\n`
- Множественные пробелы → один пробел
- Множественные пустые строки → одна пустая строка

**Правило:** в теле письма не более 1 пробела подряд, не более 1 пустой строки подряд.

**Где:** `app/imap_client.py` → `strip_html()`

## FEATURE: фильтры по датам

Добавить в `export` новые опции:

| Опция | Пример | Поведение |
|---|---|---|
| `--from-date YYYY-MM-DD` | `--from-date 2026-07-01` | Письма начиная с даты (включительно) |
| `--to-date YYYY-MM-DD` | `--to-date 2026-07-20` | Письма до даты (включительно) |
| `--before YYYY-MM-DD` | `--before 2026-07-15` | Письма строго раньше даты |
| `--after YYYY-MM-DD` | `--after 2026-07-15` | Письма строго позже даты |

**Логика:**
- `--from-date` + `--to-date` = диапазон, `--limit` ограничивает внутри диапазона
- `--before` + `--limit 5` = 5 последних писем до даты
- `--after` + `--limit 5` = 5 первых писем после даты
- `--days` и `--from-date`/`--to-date` несовместимы — приоритет у диапазона
- Все опции совместимы с `--account`, `--translate`

**Примеры:**
```bash
python -m app export --from-date 2026-07-01 --to-date 2026-07-31
python -m app export --before 2026-07-10 --limit 5
python -m app export --after 2026-07-20 --limit 10 --translate
```
